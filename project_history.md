# 자음어(ㅈㅇ) 프로젝트 개발 이력

이 문서는 자음어 언어 프로젝트의 개발 이력과 주요 마일스톤을 기록합니다.

## 1. 프로젝트 초기화 및 인터프리터 (Python)
**목표**: 파이썬을 사용하여 자음어의 작동 가능한 구현체 완성.
-   **설정**: 프로젝트 구조(`jaeum/`, `tests/`) 및 진입점(`main.py`) 생성.
-   **렉서(Lexer)**: 한글 자음과 기호를 토큰화하는 `jaeum/lexer.py` 구현.
-   **파서(Parser)**: AST(추상 구문 트리)를 생성하는 `jaeum/parser.py` (Recursive Descent) 구현.
-   **인터프리터(Interpreter)**: 즉시 실행을 위한 `jaeum/interpreter.py` 구현.
-   **검증**: 유닛 테스트 및 `test_script.jm` 생성 및 확인.

## 2. 네이티브 컴파일러 구현 (x64 Assembly)
**목표**: 자음어 코드를 실제 윈도우 실행 파일(`.exe`)로 컴파일.
-   **컴파일러 백엔드**: AST를 순회하며 NASM x64 어셈블리를 생성하는 `jaeum/compiler.py` 제작.
-   **빌드 시스템**: `Compiler` -> `NASM` -> `GoLink` 과정을 자동화하는 `build.py` 제작.
-   **도구 설정**: 빌드 의존성을 다운로드하는 `setup_tools.py` 구현.
-   **결과**: 파이썬 런타임 의존성 없이 `test_script.exe` 생성 성공.

## 3. 독립 실행형 패키징
**목표**: *컴파일러 도구 자체*의 파이썬 의존성 제거.
-   **진입점**: CLI 래퍼인 `jaeumc.py` 생성.
-   **패키징**: **PyInstaller**를 사용하여 `jaeumc.py`를 `jaeumc.exe`로 변환.
-   **독립성**: `jaeumc.exe`와 `tools/` 폴더만 있으면 파이썬 없는 환경에서도 컴파일 가능.

## 4. 언어 기능 확장 및 리팩토링
**목표**: 자체 호스팅(Self-Hosting)을 위한 언어 문법 및 기능 개선.
-   **확장자**: `.jw`에서 `.jm`으로 변경.
-   **배열(Array)**: `[1, 2]`, 인덱싱 `arr[0]`, 할당 `arr[0]=1` 문법 추가.
-   **파일 입출력(File I/O)**: 키워드 `ㅍㅇㅊㄹ`(파일쓰기), `ㅍㅇㅇㄹ`(파일읽기) 추가.
-   **표준 라이브러리 지원**: `malloc`, `fopen` 등을 활용한 네이티브 코드 생성 로직 구현.
-   **키워드 업데이트**:
    -   `ㅋ` -> `ㄹㄹ` (광역 루프 제어 - Continue)
    -   `ㅍㅌ` -> `ㅊㄹ` (출력 - Print)
    -   `브레이크(Break)`, `컨티뉴(Continue)` AST 및 파서 로직 추가.

## 5. 부트스트래핑 (현재 단계)
**목표**: 자음어 컴파일러를 **자음어 자체로** 다시 작성 (`compiler.jm`).
-   **요구사항**: 배열, 파일 입출력, 문자열 처리 기능 완비 (완료).
-   **다음 단계**: `compiler.jm`에서 렉서, 파서, 코드 생성기 구현 시작.
